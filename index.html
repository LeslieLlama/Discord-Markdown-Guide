Introduction

Markdown, or as i've called it here "text flair" is a way to modify types text within the chat program Discord, in a simmilar manner to html markdown tags. Unfortunatly they use different symbols and tags to achive the same result, ideally this guide, and their offical documentaiton, should help clairfy how to achive clear and fun text formatting within the program.

Text Size

Text size can be altered with the pound key and a space before the text (# text). This makes it appear like larger and bolded, simmilar to the numbered h tags for headers in html. You can have three different sizes of headers, all acsess by adding more pound keys, size illustrated below

Medium Big (## text)

Small Big (### text)

Something that's important to note is because these are treated like headers, they must be at the start of a new line, and cannot be blended into text. Subtext also follows this rule.

Subtext is making text smaller than the standard text size within discord, and is activated by prefacing the text with the minus key combined with a pound, like so (-#text). This will have the effect of making the text appear smaller and in a grey font on dark mode, like below.

Masked Links

A fancy way to display links is by embedding them within text akin to the a tag in html. you can do this by encircling the text you want to embed the link into with square brackets, and encircling the link within round brackets adjacent to the round brackets. [like so](link_text).

Lists

A feature you may have unintentionally used before is bulleted lists, uniquely they're made with a choice of two characters, backdash - and aesterisk *. I'd reccomend using backdash, as aesterisks are already used for bolding and italicising text. They're created by prefacing the text with backdash or aesterisk and a space, like so: (- text). You'll want to do this for each item in the list on a new line.

Lists can also be indented to create sub-lists, this is done by prefacing the text with two spaces before aesterisk or backdash ( - text) and must be done inside an already existing list
